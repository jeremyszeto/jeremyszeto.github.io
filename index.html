about me

I'm currently a third year Computer Engineering major at California Polytechnic State University, San Luis Obispo. I am comfortable coding in C, Java, and Python, and I also have experience with the hardware description language SystemVerilog as well as ARM, ARM Thumb, and RISC-V assembly syntaxes.

RISC-V Etch-A-Sketch

For a computer design class, I modeled a RISC-V microcontroller using SystemVerilog. This process involved creating and testing individual modules (memory, registers, PC, ALU, decoder, FSM, etc.) via simulation before physically programming the architecture on a Xilinx BASYS3 FPGA development board. I expanded the basic functionality of the microcontroller by programming an Etch-A-Sketch tool in C, implementing an interrupt service routine for the reset function and continuously polling for directional inputs via button presses. The Etch-A-Sketch also had options for erasing, changing the cursor speed, and changing the drawing color via switches, with visual feedback given via the onboard seven-segment display.

Where's Wheezer

For SBHacks VI, my team and I created an interactive version of Where's Waldo using Google Maps Street View API. Players navigated around a geofence where Waldo was randomly located by using the arrow keys, and would "win" the game when they approached a certain radius of Waldo's location. We wrote a computer vision algorithm to scale Waldo's size and position relative to the player's location and zoom level. You can play the game here: (unless we've run out of hosting credits...)
Our game won Best Entertainent Hack and was a Grand Prize finalist.

REX

Rex was a project I worked on with my team at SLO Hacks 2020. One of the issues that many international students face is the high fee incurred by currency transactions from their native currency to USD. We used Xpring's SDK and XRP's interledger API to smoothly transition through the transaction process from a person's bank account into XRP cryptocurrency, then into their currency of choice. Rex allows users to bypass high curency exchange rates by using blockchain technology.

C Shell

C Shell was the summative assignment for my Systems Programming class. I programmed a basic Unix shell using C89, and implemented pipelining for up to 20 commands, with fork-exec calls and file I/O for each command. Every command could also take multiple arguments and flags using dynamically allocated memory. I also included some shell built-in commands (cd, echo, exit, and pwd). Errors were checked and exeptions were handled gracefully.
